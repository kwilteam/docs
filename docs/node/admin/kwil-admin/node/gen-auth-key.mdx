Generate a new key pair for use in the node's validator set.
	
The key pair is generated and stored in the node's configuration directory, in the files auth.key and auth.cert. The key pair is used to authenticate the admin tool to the node.

### Usage

```
kwil-admin node gen-auth-key [flags]
```

### Example

```
# Generate a new TLS key pair to talk to the node
kwil-admin node gen-auth-key --authrpc-cert "~/.kwild/rpc.cert"
```

### Options

```
      --authrpc-cert string   kwild's TLS certificate
  -h, --help                  help for gen-auth-key
      --tlscert string        kwil-admin's TLS certificate file for server to authenticate us [default: auth.cert] (default "auth.cert")
      --tlskey string         kwil-admin's TLS key file to establish a mTLS (authenticated) connection [default: auth.key] (default "auth.key")
```
