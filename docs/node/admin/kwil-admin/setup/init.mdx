The `init` command facilitates quick setup of an isolated Kwil node on a fresh network in which that node is the single validator.
This permits rapid prototyping and evaluation of Kwil functionality. An output directory can be specified using the `--output-dir`" flag.
If no output directory is specified, the node will be initialized `./testnet`.

### Usage

```
kwil-admin setup init [flags]
```

### Example

```
# Initialize a node, with a new network, in the directory ~/.kwil-new
kwil-admin setup init -o ~/.kwild-new
```

### Options

```
  -i, --block-interval duration   shortest block interval in seconds (timeout_commit) [default: 6s] (default 6s)
      --chain-id string           chain ID to use for the genesis file (default: random)
  -h, --help                      help for init
      --join-expiry int           number of blocks before a join request expires [default: 14400] (default 14400)
  -o, --output-dir string         generated node parent directory [default: ./testnet] (default "./testnet")
      --without-nonces            disable account nonces
```

  -S, --silence       Silence logs
